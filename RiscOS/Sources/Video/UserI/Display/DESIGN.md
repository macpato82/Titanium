# DisplayManager — assembler→C port design

**Status: COMPLETE (build-verified, v0.45).** All six steps are ported and build
clean as a `CModule`; the 12 assembler `s/` files and `VersionASM` have been
removed (preserved in git history). The one deliberately un-ported piece is the
standalone (`-DSTANDALONE`) RAM build's ResourceFS registration (was `s/ResFiles`);
the ROM build serves resources via the ROM resource filing system. Remaining:
on-hardware confirmation in a full ROM build.

**Component:** `Video/UserI/Display` (module title `DisplayManager`, target `Display`)
**Original source:** pure ARM assembler, 12 files in `s/`, ~4,285 lines, Acorn 1996, Apache-2.0.
**Port type:** in-place, behaviour-preserving asm→C rewrite. **Apache-2.0 retained** (derivative of Acorn's code). New header (`cmhg/modhead`, processed by **CMunge** — a cmhg superset, now the tree-wide build tool) replaces the hand-written asm module header; build switched `AAsmModule`→`CModule`. CMunge gives us `module-is-runnable` and native `error-base`/`error-identifiers`.
**Version:** ported from 0.44 (29 May 2016) to **0.45** (19 Jun 2026) — behaviour identical; the bump marks the language change.

> This is a whole-module rewrite of the same class as the Territory C rewrite —
> a multi-step, build-and-verify effort, **not** a veneer tidy like the Toolbox
> components. The plan below ports file-by-file so each step builds.

---

## 1. What the module is

A Wimp application module: provides the `*Desktop_DisplayManager` command, starts
as a Wimp task at desktop start, shows an icon-bar icon (sprite `display`), and
lets the user pick a screen mode (resolution / colours / frame rate) via menus
and a dialogue. It applies the chosen mode with a `WimpMode` command.

It exports **no SWIs**. It is driven entirely by service calls (to start/stop the
task and react to mode/palette/calibration/messages-file changes) and by the Wimp
poll loop once running.

### Build options (from `s/GetAll`) — preserve as compile-time `#define`s

| Option | Meaning | Default in this tree |
|--------|---------|----------------------|
| `standalone` | register ResourceFS files (vs ROM resources) | OFF (ROM build) |
| `Medusa` | use `OS_ScreenMode` (vs old `OS_Byte 135`) | ON |
| `SelectFrameRate` | frame-rate menu + rate icon | ON |
| `LoadModeFiles` | accept DataLoad/DataSave of mode files | ON |
| `SortOnPixelShape` | sort modes by pixel shape (classes) | ON |
| `MessageChanging` | handle Service_MessageFileClosed | **OFF** |
| `Allow24bpp` | allow packed 24bpp modes | **OFF** |

Values above are verified against `s/GetAll` (this tree). `standalone` is OFF
(ROM build). Mirror them in a `h/options` header so the C compiles the same
feature set; keep the disabled branches behind `#if` so they are easy to enable.

**unchanged** — same tokens, same template names. `Makefile` keeps
`INSTRES_FILES=Templates`, `INSTRES_VERSION=Messages`.

## 3. Module state

The asm keeps one workspace block (mapped in `s/Front`, claimed at
Service_StartWimp). In C this becomes a single **statically-allocated struct**
(global; the SCL relocates it — do **not** stash it in the private word). The
private-word still identifies the instance for OS_Claim etc.

```c
typedef struct display_state {
    int   task_handle;        /* 0 = not running as task            */
    void *private_word;       /* module private word (for SWIs)     */
    int   pollword;           /* set by service calls; see pf_*     */

    /* window / icon handles */
    int   icon_handle, info_handle, mode_handle, display_handle, menu_handle;

    /* indirected window data block + sub-pointers/sizes            */
    char *indirected_data;
    char *mode_indirect;        int mode_size;
    char *resolution_indirect;  int resolution_size;
    char *colours_indirect;     int colours_size;
    char *rate_indirect;        int rate_size;      /* SelectFrameRate */
    char *title_indirect;       int title_size;

    /* mode data (separately claimed block, see c/mode)             */
    void *mode_space;
    mode_desc **mode_sortedlist;   /* all modes, sorted, NULL-term  */
    mode_desc ***mode_classlist;   /* one entry per resolution      */
    mode_desc ****mode_menulist;   /* per resolution-menu item      */
    void  *mode_table;             /* raw enumeration data          */
    wimp_menu *m_resolutionmenu;   int m_resolutionsize;
    wimp_menu *m_ratemenu;         int m_ratesize;   /* SelectFrameRate */

    int   mode_count;
    mode_desc **selected_subclass;
    mode_desc  *selected_mode;
    int   scrap_ref;               /* LoadModeFiles                 */

    int   message_file_block[4];   /* MessageTrans file block       */
    int   wimp_palette[4];         /* cached palette                */

    unsigned char flags;           /* f_greylevel/messagesopen/ratemenuvalid */
    unsigned char selected_colours;/* mo_co_*                        */
    unsigned char selected_class;
    unsigned char resolution_count, class_count, rate_count, menu_tick;
} display_state;
```

Pointer-array types (`mode_desc **/***/****`) capture the three parallel lists;
keep the indirection levels exactly as the asm uses them so the
sort/group/menu-build logic ports 1:1.

Flag/poll-word bit values (preserve exactly):
`f_greylevel=1, f_messagesopen=2, f_ratemenuvalid=4`;
`pf_refreshicons=1, pf_modeinit=2, pf_settitle=4`.

## 4. Service calls (CMHG `service-call-handler`)

Handle the same set; each maps to a small C function setting pollword bits or
managing workspace:

| Service | Action |
|---------|--------|
| `Service_Reset` (&27) | clear state, Mod_ShutDown |
| `Service_StartWimp` | start task (calls Mod_Start path) |
| `Service_StartedWimp` | clear "failed" flag |
| `Service_ModeChange` | pollword \|= modeinit\|refreshicons |
| `Service_CalibrationChanged` | pollword \|= modeinit\|refreshicons |
| `Service_WimpPalette` | scanpalette, send Message_PaletteChange, pollword \|= modeinit\|refreshicons |
| `Service_MessageFileClosed` | reopen msgs, rebuild menus (MessageChanging) |
| `Service_ResourceFSStarting` | register files (standalone) |
| `Service_ModeFileChanged` | Menu_Init, refresh icon, pollword \|= modeinit\|refreshicons\|settitle |

The asm uses the Ursula fast service table (`UServTab`); CMHG generates the
equivalent. Confirm exact reason numbers from `Hdr:Services` while porting.

## 5. Wimp poll loop (the spine — `c/module`)

```c
for (;;) {
    Wimp_Poll(poll_mask, block, &st->pollword, &event);
    switch (event) {
      case wimp_OPEN_WINDOW:    window_open(block);        break;
      case wimp_CLOSE_WINDOW:   window_close(block);       break;
      case wimp_MOUSE_CLICK:    mouse_click(block);        break;
      case wimp_KEY_PRESSED:    mode_keypressed(block);    break;
      case wimp_MENU_SELECTION: menu_selection(block);     break;
      case wimp_POLLWORD_NONZERO: mod_pollword();          break;
      case wimp_USER_MESSAGE:
      case wimp_USER_MESSAGE_RECORDED: message_received(block); break;
      default: break;   /* null/redraw/enter/leave/drag/caret/scroll/ack: ignore */
    }
}
```

`poll_mask = null | pointer-change | pollword-fast-enable` (as asm). The
ignored events (0,1,4,5,7,10,11,12,14,15,16,19) just loop — preserve that.

`mod_pollword()`: if `pf_modeinit` → Mode_Init; if `pf_refreshicons` →
Icon_Refresh; if `pf_settitle` → Window_SetTitle; clear handled bits.

## 6. The core: `c/mode` (from s/Mode, ~1624 lines)

Port these in dependency order; this is where most of the work and risk sits.
Behaviour must match exactly — the sort/dedup/match rules are subtle.

**Pixel-format helpers**
- `mode_pixelformat_from_log2bpp` — old log2bpp → GVPixelFormat (NColour, ModeFlags, Log2BPP)
- `mode_is_old_pixelformat` — can a format be expressed old-style?
- `mode_pixelformat_to_menuitem` / `mode_descriptor_to_colourmenuitem` — format → `mo_co_*` (respecting greyscale flag)
- `colours_to_pixelformat[]` table — `mo_co_*` → (NColour, ModeFlags, Log2BPP)

**Current mode / info / palette**
- `mode_getcurrent` — `OS_ScreenMode ReturnMode` (Medusa) or `OS_Byte 135`
- `mode_getinfo` — resolution/format/rate from a mode number (`OS_ReadModeVariable`: XWindLimit, YWindLimit, Log2BPP, NColour, ModeFlags) or a mode specifier (≥256)
- `mode_testpalette` / `scanpalette` — PaletteV/OS_ReadPalette, R=G=B ⇒ greyscale ⇒ `f_greylevel`

**Enumerate / sort / group**
- `mode_gettable` — `OS_ScreenMode EnumerateModes`; claim sorted/class/menu lists
- `mode_sortlist` — bubble sort by (X,Y[,pixel shape]) then format/rate; dedup ignoring RGB/BGR order + alpha; build classlist (per resolution) and menulist
- `mode_comparepixelformatsandframerates`, `mode_checkidentical`, `mode_pixelshape`

**Select / find / apply**
- `mode_setselection` — from current res+depth set colours/class + selected_mode
- `mode_findsubclass` — match by colour+class with downgrade/upgrade fallback
- `mode_buildspecifier` — descriptor → mode specifier (handle or number + vars)
- `mode_setwindowicons` / `mode_setmodestring`
- `mode_changemode` → `mode_wimpcommand` (`*WimpMode <string>`), `mode_keypressed`
- `mode_loadfile` (LoadModeFiles) — `*LoadModeFile`

**Mode descriptor layouts** (preserve offsets exactly):
- old (flags=1): size, flags, xres, yres, log2bpp, framerate, name@+24
- new (flags=3): size, flags, xres, yres, GVPixelFormat@+16 (12 bytes), framerate@+28, name@+32
- specifier: flags, xres, yres, depth(-1 for format), framerate, var-array(-1 term)
- colour items: `mo_co_mono=0,grey4,grey16,colour16,grey256,colour256,4K,32K,64K,16M=9`

Model descriptors in C as `unsigned char *` with offset accessors (or a packed
struct + a `name` accessor) rather than assuming alignment — the enumeration
buffer is OS-formatted.

## 7. Menus, windows, icons, messages

- **`c/menu`** — ROM menu templates via `MessageTrans_MakeMenus` (tokens `T00..T03`, `M01/M02`, `M11..M20`, `MX1`); build resolution menu from `mode_menulist`; build rate menu on demand (SelectFrameRate); tick/untick; `menu_selection` decodes colours/resolution/rate and updates selection then `mode_findsubclass`/`mode_setwindowicons`.
- **`c/window`** — open `DisplayManager:Templates`, load `Display`/`Mode`/`Info`, claim scratch + indirected data, create windows, patch Info version (`_Version`); open/openbehind/close; `window_settitle` from `ScreenModes_ReadInfo`.
- **`c/icon`** — iconbar icon (sprite `display`, flags 0x3002, priority `WimpPriority_ModeChooser`); `icon_setstate`; `icon_refresh`. Icon indices `ic_*` as enum.
- **`c/mouse`** — iconbar (open dialogue / colours menu), display-window buttons (colours/resolution/rate menus, OK/Cancel apply/restore), mode-window OK; `set_icon_xy` for pop-up position.
- **`c/message`** — MenuWarning (build submenu: GetCurrent/GetInfo/TestPalette/SetModeString/CreateSubMenu), MenusDeleted (clear menu_handle/tick), HelpRequest (build `H..` token, reply), DataLoad/DataSave (LoadModeFiles), Quit (shutdown+exit).
- **`c/msgtrans`** — `DisplayManager:Messages` open/lookup/errorlookup; set `f_messagesopen`.
- **`c/resources`** — ResourceFS register/deregister (standalone).

Errors (`s/Errors`) → CMHG `error-identifiers` with the same tokens
(`Modes_CantStart=UseDesk`, `NoWimp`, `E00..E05`); look up text via MessageTrans
exactly as now.

## 8. Port order (each step builds)

1. **Skeleton** — `cmhg/modhead` (title, help, helptable, services, init/final), `c/module` stubs, `h/options`, `h/display`. Switch Makefile to `CModule`, `OBJS=`. Build empty module that loads.
2. **`c/msgtrans` + `c/resources`** — message file + resources.
3. **`c/window` + `c/icon`** — templates, windows, iconbar icon; task starts and shows icon.
4. **`c/mode`** — enumerate/sort/select (the big one); verify mode list correct.
5. **`c/menu`** — colours/resolution/rate menus + selection.
6. **`c/mouse` + `c/message`** — interaction + IPC; full poll loop wired.
7. **Bump to 0.45**, full ROM build, on-hardware verify (icon, menus, apply mode, info box, help, palette/mode-change reactions).

Build cadence: assembler still compiles the whole module via `s/GetAll`; the C
port can't be half-and-half, so steps 1–6 are staged in a **scratch build**
(or by keeping `s/` until the C side is complete, then flipping the Makefile).
Recommended: develop the C files alongside, keep `OBJS` growing, and only delete
`s/` + switch `ROM_SOURCE`→`OBJS` at step 1 (skeleton) so every later step is a
real build. Keep the asm in git history for reference.
